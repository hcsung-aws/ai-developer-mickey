# 문제 해결 프로토콜

## 구현 전 필수 단계

### 1. 목적 재확인
```
"이 작업의 최종 목적은 무엇인가?"
"이 방법이 목적에 가장 단순하고 직접적인가?"
```

### 2. 전제조건 검증
```
"핵심 전제조건은 무엇인가?"
"그 전제조건이 현재 충족되어 있는가?"
```
- 전제조건 미충족 시 → 구현 진행하지 말고 대안 검토

### 3. 데이터 구조 분석 (5-10분)
- 데이터 흐름과 타이밍 이해
- 근본 원인 파악 (증상이 아닌)

### 4. 부작용 분석 (5-10분)
- 어떤 코드가 영향받는가?
- 의존성이 있는가?
- 무엇이 깨질 수 있는가?

### 5. 유사 이슈 검색
```bash
grep -r "pattern" --include="*.ext"
```

### 6. 옵션 제시 (최소 2개)
```
**Option A**: [설명]
- 장점/단점/시간/리스크

**Option B**: [설명]
- 장점/단점/시간/리스크

**권장**: Option [X] - [이유]
```

### 7. 사용자 확인 후 구현

### 8. 검증 및 문서화
- 테스트 실행
- 유사 패턴 일괄 수정 확인
- 세션 로그 업데이트

## 에러 처리 프로토콜

1. **에러 로그 즉시 확인** (추측하지 말 것)
2. **근본 원인 질문**: "왜 이 에러가 발생하는가?"
3. **영향 범위 파악**: "다른 곳에도 영향을 주는가?"
4. **해결책 제시 전 원인 설명**

## Tool/Solution Selection

새 도구/솔루션 도입 전:

1. **목적 명확화**: 이 도구가 목적에 필수적인가?
2. **복잡도 평가**: 설정 시간, 의존성, 알려진 이슈
3. **대안 검토**: 더 단순한 대안? 직접 구현이 더 빠른가?
4. **결정**: 복잡도 대비 효용 충분한가? 실패 시 대안은?

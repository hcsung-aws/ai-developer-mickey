# AI 에이전트 설계 패턴

## Context Window 절약
무거운 처리는 스크립트로 위임하고, 에이전트는 결과만 읽는다. 에이전트가 직접 대량 데이터를 처리하면 context window를 불필요하게 소모한다.

## 시간 기반 트리거의 한계
"30분마다", "1시간마다" 같은 시간 기반 트리거는 AI가 시간을 추적할 수 없으면 사문화된다. 이벤트 기반 트리거(파일 변경, 작업 완료 등)로 대체해야 한다.

## 계획 문서의 구체성 = 실행 속도
IMPROVEMENT-PLAN을 상세하게 작성해두면 구현 시 판단 비용이 거의 없다. 계획 단계에서 옵션/결정/근거를 충분히 기록할수록 실행 단계가 빨라진다.
